Cliente Web SGiS (Sistema de Gestin de Incidencias en Salud).

Descripcin y contexto

Una de las prioridades de la Secretara de Salud del Estado de Chiapas es tener herramientas para implementar acciones que permitan el acceso de las mujeres a los servicios obsttricos, a fin de reducir la muerte materna y neonatal; Por lo tanto, la creacin de SGiS, es una estrategia para fortalecer la sistematizacin en atencin de referencias, respuesta de urgencias, emergencias obsttricas y neonatales; cada unidad hospitalaria ser la central de informacin; encargada de gestionar a pacientes en salud maternal, desde el monitoreo, registro y seguimiento de las incidencias; los procesos sern controlados a travs de SGiS, estos para coordinar la red de servicios a fin de brindar una atencin resolutiva a los usuarios dentro del menor tiempo posible.

Para contribuir a mejorar la calidad y eficacia de los servicios de salud deber existir un sistema de referencia y respuesta que constituya el enlace entre las unidades hospitalarias operativas de los niveles de atencin que conforman la red de servicios, con el propsito de brindar a los usuarios atencin mdica integral y oportuna en las unidades, conforme al padecimiento de la paciente y la capacidad resolutiva de la unidad hospitalaria que resulten ms convenientes.

Gua de instalacin

Requisitos de Instalacin.

Software:

El desarrollo del Cliente web de SGiS se program en Angular 4

Para poder instalar y utilizar el Cliente web, deber asegurarse que su servidor cumpla con los siguientes requisitos, se dejan los links de descarga:

  • Angular CLI Hace que sea fcil crear una aplicacin que ya funciona.

  • Node.js Descargar: "Recomendado para la mayora"

  • Npm es un manejador de paquetes para el proyecto de Angular 4

  • Git es un sistema de control de versiones distribuidas de cdigo abierto y gratuito.

  • Google Chrome El navegador optimo para acceder a SGiS.

  • Visual Studio Code El procesador de texto recomendado para codificar y explorar las carpetas de acuerdo a la estructura del proyecto SGiS.

Si algo de lo anterior mencionado no se instalara correctamente, podr consultar la documentacin oficial de cada paquete de instalacin

Instalacin y Configuracin:

Una ves instalado todo lo anterior, abrimos una consola en nuestro servidor para clonar el proyecto en base al Repositorio.

Ejecutamos el siguiente comando en nuestra consola:


git clone https://github.com/goraider/SGiS_angular

Una ves clonado el proyecto, cargamos e instalamos todos los paquetes y sus dependencias, siempre y cuando estemos dentro de la carpeta del proyecto raz y ejecutando el siguiente comando:


npm install

Una ves instaladas nuestras las dependencias con el comando anterior, inicializamos nuestro proyecto, existen 2 formas de hacerlo:

  • Este comando inicializa todas las dependencias que node instalo de nuestro archivo package.json y a su ves inicia el servidor del cliente web en el puerto 4200.
npm start
  • O tambin este comando inicializa el servidor del cliente web, tambin en el puerto 4200.
ng serve

Si esta ocupando el puerto 4200 en otra aplicacin, pueden inicializar el proyecto cambiando dicho puerto de la siguiente manera:

ng serve --port 4201

Compilar y Poner en un servidor de produccin:

Para compilar y subir a produccin SGiS es necesario que el archivo environment.prod.ts tenga los mdulos del archivo JSON de la variable "MENU": al del del archivo environment.ts (Siempre y cuando se encuentren listos para produccin) y tener la variable "production":true.

Basta con abrir una consola de comandos en nuestro servidor y escribir el siguiente comando:

ng build --aot --env prod
